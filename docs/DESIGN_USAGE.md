# DESIGN_USAGE.md

이 문서는 K-Terminal의 디자인 시스템 및 스타일 가이드라인의 세부 수립 내용을 명시합니다. 이 프로젝트는 `DESIGN.md`를 1차 기준으로 삼아 일관성 있는 금융 터미널 화면을 제공합니다.

---

## 1. 시맨틱 컬러 토큰 및 한국 금융 관례 (CRITICAL)

K-Terminal은 **한국(KR) 주식 분석 터미널의 관례**를 준수합니다.

* **상승 (Positive / Up)**: 빨간색 계열 (`--kt-positive`, `--kt-positive-text`)을 사용하여 상승 가격 및 매수 심리를 표현합니다.
* **하락 (Negative / Down)**: 파란색 계열 (`--kt-negative-text`) 또는 어두운 회색 계열을 사용하여 하락 가격 및 매도 심리를 표현합니다.

> [!WARNING]
> 미국이나 서구권 금융 UI(상승=초록, 하락=빨강)와 정반대로 배치되는 규칙입니다. 미국 주식을 렌더링할 때에도 K-Terminal의 한국어 UI 규격을 기본으로 적용하므로 컴포넌트에 직접 `#f5445a`나 `#4391ff` 등의 하드코딩 색상을 입히는 것은 금지됩니다. 반드시 `--kt-positive-text` 및 `--kt-negative-text` 토큰을 바인딩해야 합니다. 향후 다국어(Global Locale) 전환 시 해당 토큰의 컬러 매핑만 스왑(Inversion)하여 대응할 예정입니다.

---

## 2. 뎁스(Depth)와 엘리베이션(Elevation) 규칙

* **그림자 사용 금지 (Zero Box-Shadow)**: 카드나 컴포넌트에 `box-shadow` 또는 `shadow-*` 계열의 Tailwind 클래스를 일절 적용하지 않습니다.
* **배경 레이어와 외곽선 처리**: 터미널의 뎁스는 다음 세 단계의 배경 레이어와 1px 굵기의 반투명 외곽선으로만 완성합니다.
  1. 가장 깊은 기본 바닥면: Surface 100 (`#101013`)
  2. 콘텐츠 몸통 배경: Body Composite (`rgb(23, 23, 28)`)
  3. 모달 및 패널 레이어: Overlay 300 (`#202025`)
  4. 패널 경계선: Panel border outer (`rgba(239, 215, 215, 0.09)`)

---

## 3. 라운드 처리 (Corner Radius) 규칙

터미널 내부의 컴포넌트는 오직 두 종류의 곡률만을 사용합니다.

1. **Card Radius** (`8px`): 일반적인 정보 카드(`DataCard`), 패널(`Panel`), 입력 박스(`CommandInput`), 대형 제어 버튼에 적용.
2. **Pill Radius** (`32px` / `9999px`): 상태 뱃지(`StatusBadge`), 선택 탭(`PillTabs`), 칩 스위치와 같은 작고 친근한 마이크로 UI 요소에 적용.

---

## 4. 타이포그래피 및 수치 정렬

* **폰트 파일 비포함 원칙**: 용량 다이어트 및 브랜드 자산 도용 리스크를 제거하기 위해 폰트 파일을 패키지에 포함하지 않고 OS 기본 설치 폰트 및 Toss CDN 연동 시스템 폰트 순으로 폴백을 구축합니다.
* **숫자 열 세로 정렬 (Tabular Numerals)**: 주가 정보, 재무 상태 등 정보 밀도가 높은 숫자 테이블이나 데이터 세트를 다룰 때는 반드시 `tabular-nums` 클래스를 지정해 자간 너비를 일정하게 고정합니다. 이를 통해 상하 열 스캐닝 시 자리수가 맞지 않아 발생하는 시각적 혼선을 방지합니다.

---

## 5. UI 요소 내 Emoji 금지 원칙

* **원칙**: 화면 상의 아이콘, 탭, 라벨, 정보 표시 영역에 시스템 이모지(예: 📈, 📉, 🚨, ❌)를 삽입하는 것을 절대 금지합니다.
* **이유**: 운영체제별로 이모지 렌더링 형태가 다르고 데이터 고밀도 터미널이 가져야 하는 진중한 UI 통일성을 저해하기 때문입니다.
* **대용**: `lucide-react` 단일 아이콘 라이브러리의 SVG 아이콘만을 사용하며, 색상은 `currentColor`를 상속받아 유연하게 제어합니다.
