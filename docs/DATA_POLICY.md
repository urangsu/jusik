# DATA_POLICY.md

이 문서는 K-Terminal에서 가공 및 노출하는 금융 정보와 투자 보조 지표에 대한 데이터 안전 운용 정책을 수정 불가능한 원칙으로 명시합니다.

---

## 1. 가짜 수치 노출의 원칙적 금지 (Anti-Mocking)

* **원칙**: 연동되지 않은 API나 실패한 호출에 대해 임의의 숫자(예: Mock 주가, 가상의 PER 지표 등)를 실제 주가처럼 표시하는 행위를 절대 금지합니다.
* **이유**: 주식 투자 터미널의 특성상, 개발 단계의 임시 Mock 값이 사용자에게 인지되어 잘못된 투자 판단의 원인이 될 수 있습니다.
* **대응**: 데이터 통신 실패 또는 API 미연결 시, 값(`value`)은 반드시 `null`로 보존되어야 하며, 컴포넌트는 이를 `0`이나 가짜 숫자로 바꾸어 출력해서는 안 됩니다.

---

## 2. 모든 데이터의 상태화 (Data Status)

모든 금융 데이터는 상태값(`DataStatus`)을 수반하며, 상태는 아래 규격 중 하나에 명확히 매핑되어야 합니다.

* `real_time`: 실시간 수신된 가격 데이터.
* `delayed`: 라이선스 제한으로 15분 이상 지연된 가격 데이터 (반드시 지연 뱃지를 노출해야 함).
* `eod`: 전일 종가 또는 지난 영업일 종가.
* `cached`: 네트워크 절약을 위해 브라우저 또는 서버에 임시 캐싱된 데이터.
* `api_required`: 인증 토큰이나 API 키가 인입되지 않아 요청을 수행할 수 없는 기본 비활성 상태.
* `rate_limited`: 초당/일일 호출 횟수 제한에 도달함.
* `not_supported`: 지원하지 않는 국가의 시장이나 자산군.
* `not_found`: 해당 티커/심볼이 거래소나 정보 제공사에 없음.
* `error`: 알 수 없는 연결 거부 또는 서버 장애 오류.

---

## 3. 설명자(Explainer)로서의 AI 역할 정의

* **원칙**: AI는 투자 보조 설명자(Narrator) 또는 공시 번역가(Translator)로만 제한되며, **스스로 수치 정보나 목표 주가, 매수/매도 결정을 생성(Generation)하지 않습니다.**
* **상세 규정**:
  1. AI 뷰어 컴포넌트는 수치 값을 임의로 가공하거나 지어낼 수 없습니다.
  2. "AI 의견"은 공시 본문의 요약, 팩터 점수의 수치 해석 등으로만 출처를 명기하여 텍스트로 보존합니다.
  3. AI 판단에 의한 매수/매도 추천 행위는 시스템 내부 엔진에서 수행하지 않습니다.

---

## 4. 거래 기능의 철저한 비활성화 (Order Isolation)

* **원칙**: 당 터미널은 분석 전용 화면으로, 어떠한 형태의 실거래 전송(Order Placement) 기능도 허용하지 않습니다.
* **상세 규정**:
  1. 모의 매매 인터페이스를 그리거나 시각 디자인을 확인할 경우, 해당 컨테이너는 즉시 차단 처리되어야 하고 거래 불가 상태와 그 이유가 화면상에 잠금(`Lock`) 레이아웃과 문구로 명확히 표시되어야 합니다.
  2. 실거래 API Adapter의 코드 작성을 금지하며, 버튼 클릭에 의한 가상의 주문 전송 모의 통과 창 같은 눈속임 컴포넌트(Fake UI)를 작성하지 않습니다.
