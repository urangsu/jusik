# Data Snooping Policy

과거 데이터를 반복해서 고문해 우연히 좋아 보이는 전략을 찾는 것을 방지하는 정책.

---

## 1. 데이터 스누핑이란

동일 데이터셋으로 여러 전략/파라미터를 테스트하면서, 우연히 좋은 성과를 보인 조합을
"검증된 전략"으로 착각하는 현상.

검정 횟수가 많아질수록 우연한 성과의 확률이 높아진다.
시뮬레이션이 실제 예측력을 가지려면 아래를 충족해야 한다:
- 독립적인 OOS 기간
- 전략 변형 수의 제한
- rejected 전략의 기록 유지

---

## 2. K-Terminal 데이터 스누핑 방지 규칙

### 2.1 Variant Count 제한

같은 `strategyId`에 변형(variant)이 **5개 이상**이면 자동으로 `data_snooping_possible` 경고.

이 경고는 전략을 차단하지 않는다. 하지만 연구 기록에 명시적으로 남긴다.

### 2.2 parameterHash 중복 차단

동일한 파라미터 조합의 실험을 반복하는 것을 차단한다.
`strategyId + parameterHash` 조합이 존재하면 409 응답.

### 2.3 OOS 기간 최소 요건

OOS 기간이 **6개월 미만**이면 `insufficient_oos_period` 경고.

Walk-forward 백테스트에서 총 데이터 기간의 1/3 이상이 OOS에 쓰여야 한다.

### 2.4 rejected 전략 보존

반려된 전략을 삭제하지 않는다.
rejected 전략이 기록되지 않으면, 최종적으로 선택된 전략의 "실제 검정 횟수"를 알 수 없다.

---

## 3. 체크리스트

전략 연구 보고 시 포함해야 할 항목:

- [ ] 시험한 전략 변형 총 수
- [ ] rejected 전략 수
- [ ] 데이터 스누핑 경고 발생 여부
- [ ] OOS 기간 길이
- [ ] 최종 선택 전략의 bias 경고 목록
